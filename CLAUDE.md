# 求职市场趋势分析插件（Boss方向）开发流程文档

## 1. 目标与原则
本流程用于指导 MVP 从 0 到 1 的开发与协作，核心原则：
1. 先闭环、后扩展：先打通“列表页30条->聊天回答”。
2. 先结构化、后生成：AI只消费聚合结果与少量样本。
3. 先可观测、后优化：每一步都能定位错误。

---

## 2. 研发阶段拆解

### 阶段 0：项目初始化（0.5 天）
**目标**
- 建立 React + Vite + TypeScript 的 Chrome Extension(MV3) 工程。
- 配置 Side Panel、Content Script、Background Service Worker。

**产出**
- 可运行的插件骨架。
- 基础目录结构与编码规范。

### 阶段 1：页面接入层（1~2 天）
**任务**
- URL与页面类型识别。
- 登录态识别（只读，不做自动登录）。
- 列表页职位卡片抓取（最多30条）。

**完成定义（DoD）**
- 点击“开始分析”后可拿到 rawJobs[]。
- 解析失败会返回错误码（如 `NO_JOB_CARD_FOUND`）。

### 阶段 2：标准化层（1~2 天）
**任务**
- 实现薪资/经验/学历/规模/融资阶段规范化。
- 建立技能词典与关键词抽取规则。

**完成定义（DoD）**
- 输出 normalizedJobs[]。
- 字段缺失统一为 null 并附来源状态。

### 阶段 3：统计层与 JDHS（1 天）
**任务**
- 计算薪资统计（mean/median/P25/P75）。
- 生成各维度分布。
- 计算 JDHS 与因子解释。

**完成定义（DoD）**
- 输出 aggregates 对象。
- 指标计算函数具备单元测试。

### 阶段 4：AI问答层（1 天）
**任务**
- 封装云端 API 客户端。
- 设计提示词模板（系统提示+任务提示+输出格式约束）。
- 实现回答安全护栏（不足数据时拒答/降级）。

**完成定义（DoD）**
- 能对“上海Java后端趋势”输出结构化回答。
- 回答包含样本编号引用。

### 阶段 5：Side Panel UI（1~2 天）
**任务**
- 实现状态栏、概览卡、聊天区、报告区。
- 接入流式渲染（可选）和错误提示。

**完成定义（DoD）**
- 用户在侧边栏可完成“采集->提问->看报告”闭环。

### 阶段 6：评估与测试（1 天）
**任务**
- 构建解析样例集（至少50条脱敏样例）。
- 进行字段正确率、回答一致性、性能测试。

**完成定义（DoD）**
- 输出测试报告。
- 已知问题列入 backlog。

## 4. 分支与提交流程
1. 从 `main` 切功能分支：`feat/mvp-xxx`。
2. 小步提交，提交信息遵循：`feat|fix|refactor(scope): summary`。
3. 合并前必须通过：
   - lint
   - 单元测试
   - 构建检查
4. PR 必须包含：
   - 改动说明
   - 测试结果
   - 风险与回滚方案

---

## 5. 关键开发规范

### 5.1 解析规范
- 解析函数必须纯函数化（输入HTML片段/节点，输出结构化对象）。
- 每个字段必须记录来源或兜底规则。

### 5.2 AI规范
- 禁止直接上传完整页面 HTML。
- 默认上传：聚合统计 + 少量代表样本 + 用户问题。
- 回答模板必须包含：结论、证据、建议、风险提示。

### 5.3 隐私规范
- 本地存储仅保留最近N次分析（建议N=20）。
- 样本默认脱敏公司名（可配置开关）。

### 5.4 错误处理规范
- 错误分级：`USER_ERROR / PARSE_ERROR / NETWORK_ERROR / MODEL_ERROR`。
- UI必须给出可执行提示（例如“请刷新职位列表后重试”）。

---

## 6. 测试策略

### 6.1 单元测试
- normalizer：薪资/经验/学历映射。
- analytics：均值、中位数、分位数、词频、JDHS。

### 6.2 集成测试
- content script 抓取 -> normalizer -> aggregates 数据链路。

### 6.3 回归测试
- DOM选择器快照校验，检测页面结构变化影响。

### 6.4 验收测试（手工）
- 在真实页面执行一次完整流程。
- 关键问句：
  - “帮我分析上海 Java 后端岗位趋势”
  - “主要技能要求是什么？”

---

## 7. 发布流程（内测）
1. 打包扩展并生成版本号（如 `0.1.0-beta.1`）。
2. 在 Chrome 开发者模式加载并自测。
3. 记录问题与日志，修复后发 beta.2。

---

## 8. 风险应对与迭代计划

### 8.1 风险应对
- DOM变更：解析器版本化 + 容错选择器。
- API超时：请求重试 + 本地缓存最近结果。
- 输出波动：固定输出模板 + 温度参数控制。

### 8.2 下一阶段（非MVP）
- 支持详情页抓取（模块B扩展）。
- 增加图表可视化（薪资箱线图/技能共现图）。
- 增加多岗位横向对比。

# 求职市场趋势分析插件（Boss直聘方向）需求文档

## 1. 文档信息
- 项目名称：Boss 求职市场趋势分析插件（暂定）
- 版本：v0.1（MVP 定义）
- 更新时间：2026-04-04
- 目标平台：Chrome Extension（Manifest V3）

---

## 2. 项目背景与目标
用户在 Boss 直聘页面进行职位检索时，难以快速获得结构化趋势洞察。传统浏览方式需要用户手动阅读大量职位卡片，效率低且难以形成整体判断。

本项目目标：
1. 在用户主动打开招聘页面并登录后，自动读取当前页职位数据。
2. 将原始文本解析为标准化字段并完成统计。
3. 通过 Monica 风格侧边栏支持自然语言问答。
4. 自动生成岗位洞察报告，回答如“帮我分析上海 Java 后端岗位趋势”。

---

## 3. 本期范围（MVP）

### 3.1 业务范围
- 页面范围：仅支持 **职位列表页（A）**，后续迭代再支持职位详情页（B）。
- 抓取规模：单次分析最多 **30条职位**（秒开体验优先）。
- 城市/岗位：默认从当前页面筛选条件推断（例如“上海 / Java后端”）。

### 3.2 技术范围
- 前端技术栈：React + Vite + TypeScript。
- 插件架构：保留 boss-helper 的解析思路，进行架构重构。
- AI 模式：云端 API（首版）。
- 隐私策略：默认本地聚合；仅上传脱敏统计与代表样本，不上传完整页面原文。

### 3.3 输出范围
- 支持聊天 + 报告，优先保证聊天可用。
- 报告包含：
  - 薪资统计（均值/中位数/分位数）
  - 经验分布
  - 学历分布
  - 公司规模/融资阶段分布
  - 技能词频与技能组合
  - JDHS 岗位健康度评分

---

## 4. 用户角色与核心场景

### 4.1 用户角色
- 校招/社招求职者
- 转岗用户（如前端转后端）
- 职业咨询/培训从业者

### 4.2 核心场景
1. 用户登录 Boss 直聘并进入职位列表页。
2. 打开扩展侧边栏，点击“开始分析”。
3. 插件读取最多30条职位并结构化。
4. 用户提问：“帮我分析上海 Java 后端岗位趋势”。
5. 插件输出结论与证据样本。

---

## 5. 功能需求（按模块）

## 模块 A：页面接入层
**职责**
- 识别是否处于目标站点与页面。
- 读取职位卡片、摘要信息、筛选条件（城市/关键词等）。

**输入**
- 当前 DOM、URL、登录态。

**输出**
- 原始职位记录数组（rawJobs[]）。
- 当前筛选上下文（queryContext）。

**MVP 验收标准**
- 在支持页面中解析成功率 ≥ 95%。
- 非目标页面不触发采集。

## 模块 B：数据解析与标准化层
**职责**
- 文本字段拆解并标准化。
- 异常值清洗与缺失值标记。

**标准化示例**
```json
{
  "title": "Java后端开发",
  "city": "上海",
  "salary_min_k": 20,
  "salary_max_k": 40,
  "salary_months": 14,
  "exp_level": "3-5年",
  "degree": "本科",
  "company_size": "1000-9999人",
  "funding_stage": "B轮",
  "skills": ["Java", "Spring Boot", "MySQL", "Redis"]
}
```

**MVP 验收标准**
- 薪资区间字段正确率 ≥ 95%。
- 经验/学历字段映射正确率 ≥ 95%。
- 技能提取可解释（保留命中来源片段）。

## 模块 C：统计分析层（AI前置聚合）
**职责**
- 先做本地统计聚合，再调用 AI 生成文本洞察。

**统计指标（MVP）**
- 薪资：均值、中位数、P25/P75。
- 分布：经验、学历、公司规模、融资阶段。
- 技能：词频 TopN、技能共现组合 TopN。
- JDHS：岗位健康度评分。

**JDHS（v0.1）建议公式**
- 薪资竞争力（40%）：按岗位样本分位表现。
- 门槛友好度（20%）：经验/学历要求越高分越低。
- 技术现代度（20%）：主流技术栈覆盖度。
- 公司稳定性（20%）：规模与融资阶段综合得分。

最终将总分映射到 0-100，并附可解释因子。

## 模块 D：侧边栏 UI 层
**职责**
- 提供 Monica 风格右侧面板。
- 展示采集概览、问答、报告。

**页面组成（MVP）**
1. 顶部状态区：页面识别状态、样本数量、最近分析时间。
2. 概览区：关键指标卡片。
3. 聊天区：输入框 + 对话气泡。
4. 报告区：可折叠章节（结论、指标、样本引用）。

## 模块 E：AI问答与报告层
**职责**
- 将聚合结果 + 代表样本交给模型，生成可读回答。
- 强制输出“结论 + 证据 + 风险提示”。

**回答策略**
- 不直接喂整页原文。
- 对无法判断的问题明确“不足数据”。
- 输出中引用样本编号（如 Job#03、Job#12）。

## 模块 F：评估与测试层
**职责**
- 评估解析与回答质量，保证稳定性。

**MVP 检查项**
- DOM解析成功率。
- 字段标准化正确率。
- AI回答字段一致性（避免错引字段）。
- 报告稳定性（同批输入多次生成差异可控）。

---

## 6. 非功能需求
1. 性能：30条样本全流程（采集+解析+统计+首答）目标 < 5 秒（不含API网络波动）。
2. 可维护性：解析规则与映射字典可配置化。
3. 可观测性：记录解析失败原因与AI调用错误码。
4. 安全与隐私：默认本地处理原始文本；上传前脱敏。

## 7. 数据与接口设计（MVP草案）

### 7.1 前端内部数据流
`Content Script -> Parser -> Normalizer -> Aggregator -> Side Panel Store -> AI Client -> Renderer`

### 7.2 AI请求体草案
```json
{
  "meta": {
    "site": "boss",
    "city": "上海",
    "keyword": "Java后端",
    "sample_size": 30
  },
  "aggregates": {
    "salary": {"mean_k": 28.5, "median_k": 27, "p25_k": 22, "p75_k": 35},
    "exp_dist": {"1-3年": 0.3, "3-5年": 0.45, "5-10年": 0.2, "不限": 0.05},
    "degree_dist": {"大专": 0.1, "本科": 0.75, "硕士": 0.15},
    "skills_top": [{"skill": "Java", "count": 28}, {"skill": "Spring Boot", "count": 22}]
  },
  "samples": [
    {"id": "Job#01", "title": "Java后端开发", "salary": "25-35K·14薪", "skills": ["Java", "MySQL"]}
  ]
}
```

## 8. 风险与边界
1. DOM结构变更风险：通过选择器版本化 + 兜底策略降低故障。
2. 反爬/风控风险：仅用户主动触发、低频采集、限制样本量。
3. AI幻觉风险：仅基于结构化统计作答，必须附样本引用。
4. 口径风险：在报告中固定指标定义与统计口径。

---

- M1：完成页面接入与30条采集（列表页）。
- M2：完成标准化与统计模块。
- M3：完成Side Panel聊天与报告渲染。
- M4：完成评估用例与稳定性修复，发布内测版。

## 10. 验收标准（MVP）
1. 在目标页面可稳定抓取并完成分析，失败可回退提示。
2. 用户提问“帮我分析上海 Java 后端岗位趋势”可得到结构化结论与样本引用。
3. 指标覆盖薪资/经验/学历/公司规模融资/技能/JDHS。
4. 默认隐私策略生效：原始职位文本不直接上传。
