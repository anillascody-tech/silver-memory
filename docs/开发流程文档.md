# 求职市场趋势分析插件（Boss方向）开发流程文档

## 1. 目标与原则
本流程用于指导 MVP 从 0 到 1 的开发与协作，核心原则：
1. 先闭环、后扩展：先打通“列表页30条->聊天回答”。
2. 先结构化、后生成：AI只消费聚合结果与少量样本。
3. 先可观测、后优化：每一步都能定位错误。

---

## 2. 研发阶段拆解

### 阶段 0：项目初始化（0.5 天）
**目标**
- 建立 React + Vite + TypeScript 的 Chrome Extension(MV3) 工程。
- 配置 Side Panel、Content Script、Background Service Worker。

**产出**
- 可运行的插件骨架。
- 基础目录结构与编码规范。

### 阶段 1：页面接入层（1~2 天）
**任务**
- URL与页面类型识别。
- 登录态识别（只读，不做自动登录）。
- 列表页职位卡片抓取（最多30条）。

**完成定义（DoD）**
- 点击“开始分析”后可拿到 rawJobs[]。
- 解析失败会返回错误码（如 `NO_JOB_CARD_FOUND`）。

### 阶段 2：标准化层（1~2 天）
**任务**
- 实现薪资/经验/学历/规模/融资阶段规范化。
- 建立技能词典与关键词抽取规则。

**完成定义（DoD）**
- 输出 normalizedJobs[]。
- 字段缺失统一为 null 并附来源状态。

### 阶段 3：统计层与 JDHS（1 天）
**任务**
- 计算薪资统计（mean/median/P25/P75）。
- 生成各维度分布。
- 计算 JDHS 与因子解释。

**完成定义（DoD）**
- 输出 aggregates 对象。
- 指标计算函数具备单元测试。

### 阶段 4：AI问答层（1 天）
**任务**
- 封装云端 API 客户端。
- 设计提示词模板（系统提示+任务提示+输出格式约束）。
- 实现回答安全护栏（不足数据时拒答/降级）。

**完成定义（DoD）**
- 能对“上海Java后端趋势”输出结构化回答。
- 回答包含样本编号引用。

### 阶段 5：Side Panel UI（1~2 天）
**任务**
- 实现状态栏、概览卡、聊天区、报告区。
- 接入流式渲染（可选）和错误提示。

**完成定义（DoD）**
- 用户在侧边栏可完成“采集->提问->看报告”闭环。

### 阶段 6：评估与测试（1 天）
**任务**
- 构建解析样例集（至少50条脱敏样例）。
- 进行字段正确率、回答一致性、性能测试。

**完成定义（DoD）**
- 输出测试报告。
- 已知问题列入 backlog。

## 4. 分支与提交流程
1. 从 `main` 切功能分支：`feat/mvp-xxx`。
2. 小步提交，提交信息遵循：`feat|fix|refactor(scope): summary`。
3. 合并前必须通过：
   - lint
   - 单元测试
   - 构建检查
4. PR 必须包含：
   - 改动说明
   - 测试结果
   - 风险与回滚方案

---

## 5. 关键开发规范

### 5.1 解析规范
- 解析函数必须纯函数化（输入HTML片段/节点，输出结构化对象）。
- 每个字段必须记录来源或兜底规则。

### 5.2 AI规范
- 禁止直接上传完整页面 HTML。
- 默认上传：聚合统计 + 少量代表样本 + 用户问题。
- 回答模板必须包含：结论、证据、建议、风险提示。

### 5.3 隐私规范
- 本地存储仅保留最近N次分析（建议N=20）。
- 样本默认脱敏公司名（可配置开关）。

### 5.4 错误处理规范
- 错误分级：`USER_ERROR / PARSE_ERROR / NETWORK_ERROR / MODEL_ERROR`。
- UI必须给出可执行提示（例如“请刷新职位列表后重试”）。

---

## 6. 测试策略

### 6.1 单元测试
- normalizer：薪资/经验/学历映射。
- analytics：均值、中位数、分位数、词频、JDHS。

### 6.2 集成测试
- content script 抓取 -> normalizer -> aggregates 数据链路。

### 6.3 回归测试
- DOM选择器快照校验，检测页面结构变化影响。

### 6.4 验收测试（手工）
- 在真实页面执行一次完整流程。
- 关键问句：
  - “帮我分析上海 Java 后端岗位趋势”
  - “主要技能要求是什么？”

---

## 7. 发布流程（内测）
1. 打包扩展并生成版本号（如 `0.1.0-beta.1`）。
2. 在 Chrome 开发者模式加载并自测。
3. 记录问题与日志，修复后发 beta.2。

---

## 8. 风险应对与迭代计划

### 8.1 风险应对
- DOM变更：解析器版本化 + 容错选择器。
- API超时：请求重试 + 本地缓存最近结果。
- 输出波动：固定输出模板 + 温度参数控制。

### 8.2 下一阶段（非MVP）
- 支持详情页抓取（模块B扩展）。
- 增加图表可视化（薪资箱线图/技能共现图）。
- 增加多岗位横向对比。
